# Claude Code Setup Guide

## Step 1: Install DSAEngine

### Option A: Using Cargo (requires Rust)
```bash
cargo install dsaengine
```

### Option B: Download Pre-built Binary
1. Go to [GitHub Releases](https://github.com/yourusername/dsaengine/releases)
2. Download `dsaengine.exe` (Windows) or `dsaengine` (macOS/Linux)
3. Move to a directory in your PATH, or note the full path

### Option C: Build from Source
```bash
git clone https://github.com/yourusername/dsaengine
cd dsaengine
cargo build --release
# Binary: target/release/dsaengine.exe (Windows) or target/release/dsaengine (macOS/Linux)
```

---

## Step 2: Configure Claude Code MCP Settings

Edit `~/.claude/claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "dsaengine": {
      "command": "dsaengine",
      "args": ["--mcp"],
      "description": "DSAEngine — 142+ optimal DSA algorithm implementations"
    }
  }
}
```

**File location:**
- **Windows**: `%USERPROFILE%\.claude\claude_desktop_config.json`
- **macOS**: `~/.claude/claude_desktop_config.json`
- **Linux**: `~/.claude/claude_desktop_config.json`

If the file doesn't exist, create it with the above JSON.

---

## Step 3: Restart Claude Code

1. Close Claude Code completely
2. Reopen Claude Code
3. You should see a notification that DSAEngine MCP is connected

---

## Step 4: Start Coding

Ask Claude to write code involving algorithms:
- "Write a function to find the shortest path in a weighted graph"
- "Implement a median finder that supports adding numbers"
- "Create a word ladder solver"

Claude will automatically consult DSAEngine for the **optimal algorithm** before writing code.

---

## Verification

To verify DSAEngine is working:

1. In the Claude interface, ask: "What tools do you have access to?"
2. You should see `dsa_classify` listed among the available tools
3. Ask Claude to solve an algorithmic problem and watch it call DSAEngine

## Troubleshooting

### "stderr: spawn ENOENT"
- The `dsaengine` binary path is incorrect
- Verify the binary exists:
  ```bash
  which dsaengine   # macOS/Linux
  where dsaengine   # Windows PowerShell
  ```
- Use the full path in `claude_desktop_config.json` if not in PATH

### "Failed to initialize tool"
- Test manually:
  ```bash
  dsaengine --mcp
  # Should print: [dsaengine-mcp] stdio server ready
  ```

### "Claude not using DSAEngine"
- Verify `CLAUDE.md` exists in your project root (auto-generated by `dsaengine --install`)
- Restart Claude Code

## Tips

### Activate Project Rules
Run this in your project directory:
```bash
dsaengine --install
```

This auto-generates `CLAUDE.md` which Claude reads on every interaction.

### Test DSAEngine Directly
```bash
# Send a test classification request
echo '{"jsonrpc":"2.0","id":1,"method":"tools/call","params":{"name":"dsa_classify","arguments":{"description":"find shortest path in weighted graph"}}}' | dsaengine --mcp
```
